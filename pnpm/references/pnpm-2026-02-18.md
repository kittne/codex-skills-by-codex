## Practical Guidance
- Use `pnpm-workspace.yaml` with explicit include/exclude patterns.
- Use `workspace:` protocol for internal package dependencies.
- Enforce `pnpm install --frozen-lockfile` in CI.
- Use `--filter` selectors for targeted builds/tests in monorepos.
- Use recursive run modes for consistent cross-package scripts.

## Command Patterns
```bash
pnpm install --frozen-lockfile
pnpm -r run build
pnpm --filter "...[origin/main]" run test
pnpm dedupe
pnpm audit
```

## Operational Notes
- Prefer caching pnpm store in CI.
- Avoid lockfile mutation in protected branches.
- Keep a single package manager per repository.
