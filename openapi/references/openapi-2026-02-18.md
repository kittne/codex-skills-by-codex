## Context7 Sources

## Practical Guidance
- Use reusable `components` for schemas, parameters, responses, and security schemes.
- Keep `operationId` stable for SDK and integration reliability.
- Use composition (`allOf`, `oneOf`) intentionally and document discriminator behavior.
- Define security schemes centrally and apply explicitly at operation level.
- Treat compatibility checks as a merge gate.

## Notable Spec Anchors
- Components object structure.
- Schema composition and polymorphism semantics.
- Security scheme definitions.
